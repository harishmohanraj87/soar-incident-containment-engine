# SOAR Event Schema Design

## Overview

This document defines the standardized event schema for the SOAR Incident Containment Engine. The schema ensures consistent logging, auditability, and incident tracking across all system components.

---

## Objectives

The event schema aims to:

* Standardize security events
* Enable incident timeline reconstruction
* Support audit logging
* Facilitate SIEM integration
* Improve forensic analysis
* Maintain compliance requirements

---

## Common Event Structure

Every event generated by the SOAR system must follow the same schema.

### Event Fields

| Field      | Type    | Required | Description                     |
| ---------- | ------- | -------- | ------------------------------- |
| event_id   | UUID    | Yes      | Unique event identifier         |
| timestamp  | ISO8601 | Yes      | Event creation time in UTC      |
| event_type | String  | Yes      | Type of security event          |
| severity   | String  | Yes      | low, medium, high, critical     |
| alert_id   | UUID    | Yes      | Associated alert identifier     |
| actor      | String  | Yes      | User or system generating event |
| status     | String  | Yes      | success or failure              |
| source     | String  | Yes      | Originating service             |
| details    | JSON    | Yes      | Additional event information    |

---

## Standard Event Types

### Alert Events

* ALERT_RECEIVED
* ALERT_NORMALIZED
* IOC_EXTRACTED

### Enrichment Events

* ENRICHMENT_STARTED
* ENRICHMENT_COMPLETED
* THREAT_LOOKUP_FAILED

### Playbook Events

* PLAYBOOK_TRIGGERED
* PLAYBOOK_COMPLETED
* PLAYBOOK_FAILED

### Containment Events

* IP_BLOCKED
* HOST_ISOLATED
* SECURITY_GROUP_UPDATED

### User Events

* USER_LOGIN
* RBAC_APPROVAL
* ACCESS_DENIED

### System Events

* ERROR
* DATABASE_FAILURE
* API_TIMEOUT

## Design Considerations

* Use UTC timestamps for consistency.
* Generate UUIDs for unique identification.
* Store logs in structured JSON format.
* Ensure immutability of audit logs.
* Support future integration with SIEM tools.

---

## Conclusion

A standardized event schema ensures reliable logging, incident traceability, and compliance with cybersecurity best practices. It serves as the foundation for audit logging and automated incident response in the SOAR platform.
